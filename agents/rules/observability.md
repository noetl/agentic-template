# Observability

Distributed runtimes are only debuggable when traceability is built into the
implementation. Pair this rule with [`logging.md`](logging.md): logging limits
noise, while this rule defines spans, metrics, and correlation identifiers.

## Principle 1: observability ships with the change

Every new feature, boundary call, dispatch path, or runtime-impacting behavior
lands with:

1. **A span** covering the operation.
2. **At least one metric** for throughput, latency, queue depth, or failure.
3. **A correlation identifier** such as `execution_id`, `request_id`, or
   `trace_id` flowing through logs/spans/events.

If a substantive PR has no observability story, the reviewer should ask where
it belongs.

## Principle 2: metrics over logs

INFO logs are not observability. Use counters for throughput, histograms for
latency, gauges for backlog/queue depth, and structured errors for failures.
When adding a log line, ask whether a metric increment plus a span event would
answer the question better.

## Principle 3: IDs come from the application when correlation needs them early

Identifiers needed before persistence or publish time should be generated by
the application, not only by a database default. This keeps spans correlated,
retries idempotent, and cross-component messages traceable.

Use a proven library for snowflake/ULID/UUID generation rather than
hand-rolling an ID algorithm.

## Principle 4: correlation rides every boundary

- HTTP requests include a request or execution identifier in path, header, or
  query where appropriate.
- Queue messages include the identifier in headers/attributes, not only the
  JSON body.
- Spans include the identifier as a field.
- WARN/ERROR logs include the identifier in structured fields.
- High-cardinality IDs should not become raw Prometheus labels unless your
  metrics backend supports that cardinality.

## What to instrument first

- gateway/service HTTP calls
- queue publish and claim paths
- worker/tool dispatch
- external API or database calls
- event-log writes
- callback/resume boundaries
